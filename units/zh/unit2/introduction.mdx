# 构建端到端 MCP 应用程序

欢迎来到 MCP 课程的第 2 单元！

在这个单元中，我们将从头开始构建一个完整的 MCP 应用程序，重点是使用 Gradio 创建服务器并将其与多个客户端连接。这种实践方法将为您提供整个 MCP 生态系统的实际经验。

<Tip>

在这个单元中，我们将使用 Gradio 和 HuggingFace hub 构建一个简单的 MCP 服务器和客户端。在下一个单元中，我们将构建一个更复杂的服务器来解决现实世界的用例。

</Tip>

## 您将学到什么

在这个单元中，您将：

- 使用 Gradio 的内置 MCP 支持创建 MCP 服务器
- 构建一个可供 AI 模型使用的情感分析工具
- 使用不同的客户端实现连接到服务器：
  - 基于 HuggingFace.js 的客户端
  - 基于 SmolAgents 的 Python 客户端
- 将您的 MCP 服务器部署到 Hugging Face Spaces
- 测试和调试完整系统

在本单元结束时，您将拥有一个工作的 MCP 应用程序，展示协议的强大功能和灵活性。

## 先决条件

在继续本单元之前，请确保您：

- 已完成第 1 单元或对 MCP 概念有基本了解
- 熟悉 Python 和 JavaScript/TypeScript
- 对 API 和客户端-服务器架构有基本了解
- 拥有开发环境：
  - Python 3.10+
  - Node.js 18+
  - Hugging Face 账户（用于部署）

## 我们的端到端项目

我们将构建一个情感分析应用程序，由三个主要部分组成：服务器、客户端和部署。

![情感分析应用程序](https://huggingface.co/datasets/mcp-course/images/resolve/main/unit2/1.png)

### 服务器端

- 使用 Gradio 通过 `gr.Interface` 创建 Web 界面和 MCP 服务器
- 使用 TextBlob 实现情感分析工具
- 通过 HTTP 和 MCP 协议公开工具

### 客户端

- 实现 HuggingFace.js 客户端
- 或者，创建 smolagents Python 客户端
- 演示如何使用不同的客户端实现使用同一服务器

### 部署

- 将服务器部署到 Hugging Face Spaces
- 配置客户端以与部署的服务器一起工作
