# 关键概念和术语

在深入了解模型上下文协议之前，了解构成 MCP 基础的关键概念和术语非常重要。本节将介绍支撑协议的基本思想，并为在整个课程中讨论 MCP 实现提供通用词汇。

MCP 通常被描述为"AI 应用程序的 USB-C"。正如 USB-C 为将各种外围设备连接到计算设备提供标准化的物理和逻辑接口一样，MCP 为将 AI 模型链接到外部能力提供一致的协议。这种标准化使整个生态系统受益：

- **用户**在 AI 应用程序中享受更简单、更一致的体验
- **AI 应用程序开发者**可以轻松集成不断增长的工具和数据源生态系统
- **工具和数据提供者**只需创建一个适用于多个 AI 应用程序的实现
- 更广泛的生态系统受益于增强的互操作性、创新和减少的碎片化

## 集成问题

**M×N 集成问题**指的是在没有标准化方法的情况下，将 M 个不同的 AI 应用程序连接到 N 个不同的外部工具或数据源的挑战。

### 没有 MCP（M×N 问题）

没有像 MCP 这样的协议，开发者需要创建 M×N 个自定义集成——每个 AI 应用程序与外部能力的可能配对都需要一个。

![没有 MCP](https://huggingface.co/datasets/mcp-course/images/resolve/main/unit1/1.png)

每个 AI 应用程序都需要与每个工具/数据源单独集成。这是一个非常复杂和昂贵的过程，为开发者带来了很多摩擦，并且维护成本很高。

一旦我们有多个模型和多个工具，集成的数量就变得太大而无法管理，每个都有自己独特的接口。

![多个模型和工具](https://huggingface.co/datasets/mcp-course/images/resolve/main/unit1/1a.png)

### 使用 MCP（M+N 解决方案）

MCP 通过提供标准接口将其转换为 M+N 问题：每个 AI 应用程序实现一次 MCP 的客户端，每个工具/数据源实现一次服务器端。这大大降低了集成复杂性和维护负担。

![使用 MCP](https://huggingface.co/datasets/mcp-course/images/resolve/main/unit1/2.png)

每个 AI 应用程序实现一次 MCP 的客户端，每个工具/数据源实现一次服务器端。

## 核心 MCP 术语

现在我们了解了 MCP 解决的问题，让我们深入了解构成 MCP 协议的核心术语和概念。

<Tip>

MCP 是像 HTTP 或 USB-C 这样的标准，是连接 AI 应用程序到外部工具和数据源的协议。因此，使用标准术语对于使 MCP 有效工作至关重要。

在记录我们的应用程序和与社区沟通时，我们应该使用以下术语。

</Tip>

### 组件

就像 HTTP 中的客户端服务器关系一样，MCP 有客户端和服务器。

![MCP 组件](https://huggingface.co/datasets/mcp-course/images/resolve/main/unit1/3.png)

- **主机（Host）**：面向用户的 AI 应用程序，最终用户直接与之交互。示例包括 Anthropic 的 Claude Desktop、AI 增强的 IDE（如 Cursor）、推理库（如 Hugging Face Python SDK）或在 LangChain 或 smolagents 等库中构建的自定义应用程序。主机启动与 MCP 服务器的连接，并协调用户请求、LLM 处理和外部工具之间的整体流程。

- **客户端（Client）**：主机应用程序内管理与特定 MCP 服务器通信的组件。每个客户端与单个服务器保持 1:1 连接，处理 MCP 通信的协议级细节，并充当主机逻辑和外部服务器之间的中介。

- **服务器（Server）**：通过 MCP 协议公开能力（工具、资源、提示）的外部程序或服务。

<Tip warning={true}>

很多内容将"客户端"和"主机"互换使用。从技术上讲，主机是面向用户的应用程序，客户端是主机应用程序内管理与特定 MCP 服务器通信的组件。

</Tip>

### 能力

当然，您的应用程序的价值是它提供的能力的总和。因此，能力是您应用程序最重要的部分。MCP 可以连接任何软件服务，但有一些常见的能力用于许多 AI 应用程序。

| 能力 | 描述 | 示例 |
| ---------- | ----------- | ------- |
| **工具（Tools）** | AI 模型可以调用以执行操作或检索计算数据的可执行函数。通常与应用程序的用例相关。 | 天气应用程序的工具可能是返回特定位置天气的函数。 |
| **资源（Resources）** | 提供上下文而无需大量计算的只读数据源。 | 研究助手可能有科学论文的资源。 |
| **提示（Prompts）** | 指导用户、AI 模型和可用能力之间交互的预定义模板或工作流程。 | 摘要提示。 |
| **采样（Sampling）** | 服务器发起的请求，要求客户端/主机执行 LLM 交互，启用递归操作，其中 LLM 可以审查生成的内容并做出进一步决策。 | 写作应用程序审查自己的输出并决定进一步完善它。 |

![集体图表](https://huggingface.co/datasets/mcp-course/images/resolve/main/unit1/8.png)

此应用程序可能以以下方式使用其 MCP 实体：

| 实体 | 名称 | 描述 |
| --- | --- | --- |
| 工具 | 代码解释器 | 可以执行 LLM 编写的代码的工具。 |
| 资源 | 文档 | 包含应用程序文档的资源。 |
| 提示 | 代码风格 | 指导 LLM 生成代码的提示。 |
| 采样 | 代码审查 | 允许 LLM 审查代码并做出进一步决策的采样。 |
